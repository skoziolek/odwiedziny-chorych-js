Przetestowanie poprawionych funkcji generowania danych testowych, które teraz zawierają pełne imiona i nazwiska chorych oraz szafarzy.

📋 Wymagania systemowe

Komponent Wymagania Status
System operacyjny Windows 10/11 ☐ Sprawdź
PHP PHP 8.0 lub nowszy ☐ Sprawdź
Przeglądarka Chrome, Firefox, Edge ☐ Sprawdź
Port 8000 (dostępny) ☐ Sprawdź

🚀 Instalacja i uruchomienie

Krok 1: Sprawdzenie PHP

Sprawdź czy masz zainstalowane PHP:

cmd
php --version

Jeśli widzisz informację o wersji PHP - masz zainstalowane. Jeśli nie - pobierz z php.net

Krok 2: Rozpakowanie aplikacji

Rozpakuj plik ZIP do folderu, np.:

C:\Odwiedziny_Chorych_Tester

Upewnij się, że wszystkie pliki są w jednym folderze.

Krok 3: Uruchomienie serwera

Uruchom plik START_APLIKACJI.bat:

  • Dwuklik na plik START_APLIKACJI.bat
  • Lub prawy przycisk myszy → "Uruchom jako administrator"

Powinno pojawić się okno z informacją o uruchomionym serwerze PHP.

Krok 4: Dostęp do aplikacji

Otwórz przeglądarkę i wejdź na:

http://localhost:8000

Powinna pojawić się strona logowania aplikacji.

Krok 5: Logowanie

Zaloguj się używając:

Hasło: PomocDlaChorych!

🧪 Procedura testowania

Test 1: Generowanie danych testowych

Sprawdź czy dane testowe zawierają pełne imiona i nazwiska:

  1. Przejdź do zakładki "Kalendarz"
  2. Kliknij przycisk "Generuj dane testowe"
  3. Potwierdź akcję w oknie dialogowym
  4. Sprawdź zakładkę "Chorzy" - czy widzisz pełne imiona i nazwiska
  5. Sprawdź zakładkę "Szafarze" - czy widzisz pełne imiona i nazwiska
  6. Sprawdź kalendarz - czy widzisz pełne imiona i nazwiska szafarzy

Test 2: Wyświetlanie danych

Sprawdź czy dane są poprawnie wyświetlane:

  • W kalendarzu powinny być pełne imiona i nazwiska szafarzy

Test 3: Czyszczenie danych testowych

  2. Potwierdź akcję w oknie dialogowym
  3. Sprawdź czy wszystkie dane zostały usunięte z wszystkich zakładek

⚠️ Rozwiązywanie problemów

Problem: "Port 8000 jest zajęty"

Rozwiązanie:

  1. Zamknij inne aplikacje używające port 8000
  2. Lub edytuj plik START_APLIKACJI.bat i zmień port na 8080
  3. Następnie wejdź na http://localhost:8080

Problem: "PHP nie jest rozpoznawane jako polecenie"

Rozwiązanie:

  1. Zainstaluj PHP ze strony php.net
  2. Dodaj PHP do zmiennej PATH systemowej
  3. Lub edytuj START_APLIKACJI.bat i wpisz pełną ścieżkę do php.exe

Problem: "Błąd szyfrowania danych"

Rozwiązanie:

  1. Sprawdź czy wszystkie pliki zostały skopiowane
  2. Uruchom ponownie serwer PHP
  3. Wyczyść dane testowe i wygeneruj ponownie
